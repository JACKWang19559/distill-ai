# Distill - AI 知识蒸馏站 产品需求文档（PRD）

> **版本**：v1.0 (MVP)
> **创建日期**：2026-06-17
> **项目类型**：2026 AI TRAE 创造力大赛参赛作品
> **状态**：已确认

---

## 一、产品概述

### 1.1 产品名称
**Distill** —— AI 驱动的个人知识蒸馏与图谱系统

### 1.2 产品定位
一款浏览器插件 + Web 端联动的个人知识管理系统，支持一键导入任意格式信息源，通过 AI 蒸馏核心要点，自动建立知识图谱关联，实现"输入即内化"。

### 1.3 产品愿景
消除"收藏即学习"的错觉，让每一条信息都能被真正消化、结构化、关联化，构建可生长的个人知识网络。

---

## 二、问题陈述

### 2.1 用户痛点
信息过载时代，用户面临三重困境：

1. **信息输入过载**：每天被海量文章、课程、会议录音、PDF 文档淹没，收藏了几百 G 资料却从未真正消化
2. **内化效率极低**：90% 的人有"收藏即学习"的错觉，真正吸收的不到 5%
3. **知识孤岛严重**：收藏的内容散落在网盘、书签、笔记各处，需要用时找不到、记不清，知识转化率不足 10%

### 2.2 现有方案不足
- **手动划重点 + 写笔记**：耗时耗力，3 小时长文需要 3 小时精读
- **直接收藏**：再也不看，信息沉睡
- **现有笔记软件**（Notion/Obsidian）：需要用户主动整理，未解决"消化"环节
- **现有 AI 总结工具**：多为一次性总结，缺乏知识库沉淀与关联

## 三、目标用户

### 3.1 用户画像

| 画像 | 占比 | 典型场景 | 核心诉求 |
|------|------|----------|----------|
| 职场白领 | 35% | 刷到优质公众号文章、行业报告 | 快速消化、留存核心观点 |
| 在校研究生 | 25% | 阅读论文、整理文献综述 | 结构化整理、建立知识关联 |
| 终身学习者 | 25% | 听完行业课程、读书笔记 | 高效内化、构建知识体系 |
| 知识工作者 | 15% | 开完 3 小时长会、整理会议纪要 | 快速提炼、可检索回溯 |

### 3.2 核心使用场景

**场景 1：公众号文章蒸馏**
> 用户在地铁上刷到一篇 5000 字的深度行业分析，点击浏览器插件"一键蒸馏"，5 秒内获得 200 字摘要 + 5 个关键要点 + 推荐标签，自动入库并关联到已有相关知识。

**场景 2：行业报告 PDF 处理**
> 用户下载了一份 50 页的行业报告 PDF，上传到 Web 端，30 秒内获得结构化大纲 + 核心数据点 + 思维导图，并自动与知识库中相关报告建立关联。

**场景 3：会议纪要提炼**
> 用户开完 3 小时长会，将会议录音转写文本粘贴到 Web 端，获得决策要点 + 行动项 + 关键讨论摘要，自动打上"会议"标签。

**场景 4：知识体系构建**
> 用户蒸馏了 20 篇 AI 相关文章后，打开知识图谱页面，看到所有知识点以可视化网络呈现，点击某个节点可查看详情，发现知识点之间的关联关系。

## 四、MVP 功能范围

### 4.1 包含功能（P0 - 必须有）

#### 功能 1：一键蒸馏 + 结构化总结
- **输入源**：
  - 网页 URL（浏览器插件一键提取）
  - PDF 文件（使用 opendataloader-pdf 引擎，支持复杂表格/OCR/公式）
  - 纯文本、Markdown
  - **抖音视频链接**（Python 服务下载 → 分离音频 → ASR 识别文案 → 蒸馏）
  - **小红书笔记/视频链接**（Python 服务提取图文或视频 → ASR → 蒸馏）
- **输出**：
  - 200 字以内的核心摘要
  - 3-7 个关键要点（每个不超过 50 字）
  - 层级大纲（最多 3 级）
  - 思维导图数据结构
  - 推荐标签（3-5 个）
- **交互**：
  - 浏览器插件：点击图标 → 侧边栏预览 → 一键蒸馏 → 流式展示结果
  - Web 端：粘贴文本/上传 PDF/粘贴抖音或小红书链接 → 蒸馏 → 查看结果 → 保存

#### 功能 2：知识库管理 + 搜索
- **管理**：
  - 知识条目列表（卡片视图/列表视图切换）
  - 详情查看（摘要 + 大纲 + 关键点 + 标签 + 原文）
  - 编辑（修改标题、标签、补充笔记）
  - 删除（软删除，可恢复）
- **标签系统**：
  - 创建标签（自定义名称 + 颜色）
  - 按标签过滤
  - 自动推荐标签（蒸馏时 AI 生成）
- **搜索**：
  - 全文搜索（标题、摘要、原文）
  - 按标签、时间范围、来源类型过滤
  - 搜索结果高亮关键词

#### 功能 3：知识图谱关联
- **自动关联**：
  - 蒸馏时抽取实体（人名/概念/组织/技术）
  - 基于实体重叠 + 标签相似度自动建立关联
  - 关联类型：related / similar / references / extends
- **可视化**：
  - 力导向布局的图谱展示
  - 节点大小反映连接数
  - 边粗细反映关联强度
  - 节点点击跳转详情
  - 按标签、时间范围过滤

### 4.2 不包含功能（留待 v2）
- 移动端 App
- 艾宾浩斯间隔复习 + 闪卡
- 多人协作
- 离线模式
- B 站/YouTube/快手等其他视频平台（MVP 仅支持抖音 + 小红书）
- AI 问答（RAG）

## 五、功能需求详细说明

### 5.1 用户认证

#### 需求：用户注册与登录
系统 SHALL 提供用户注册和登录功能，支持 GitHub OAuth 和邮箱密码两种方式。

**场景：GitHub 登录**
- **WHEN** 用户点击"使用 GitHub 登录"
- **THEN** 跳转 GitHub 授权页，授权后自动创建/登录账户并跳转到工作台

**场景：邮箱注册**
- **WHEN** 用户输入邮箱和密码点击注册
- **THEN** 系统创建账户并发送验证邮件，用户验证后可登录

**场景：未登录访问**
- **WHEN** 未登录用户访问工作台页面
- **THEN** 重定向到登录页

### 5.2 AI 供应商配置

#### 需求：多供应商 API 配置
系统 SHALL 支持用户配置多个 LLM 供应商的 API Key，并可切换默认供应商。

**支持的供应商**：
- OpenAI（GPT-4o / GPT-4o-mini）
- Anthropic（Claude 3.5 Sonnet / Haiku）
- 通义千问（Qwen-Max / Qwen-Plus）
- DeepSeek（DeepSeek-Chat / DeepSeek-Coder）
- 智谱 GLM（GLM-4 / GLM-4-Flash）

**场景：添加 API Key**
- **WHEN** 用户在设置页选择供应商并输入 API Key
- **THEN** 系统加密存储 Key，并标记为可用

**场景：切换默认供应商**
- **WHEN** 用户将某供应商设为默认
- **THEN** 后续蒸馏任务使用该供应商

**场景：API Key 验证失败**
- **WHEN** 用户输入无效 Key
- **THEN** 系统提示错误并允许重新输入

### 5.3 蒸馏功能

#### 需求：网页内容蒸馏
系统 SHALL 支持通过浏览器插件一键蒸馏当前网页内容。

**场景：正常蒸馏**
- **WHEN** 用户在网页点击插件图标并点击"一键蒸馏"
- **THEN** 系统提取页面正文，调用 AI 生成结构化总结，10 秒内返回结果

**场景：蒸馏选中内容**
- **WHEN** 用户选中网页部分文本，右键选择"蒸馏选中内容"
- **THEN** 系统仅对选中内容进行蒸馏

**场景：页面内容过少**
- **WHEN** 提取的正文少于 100 字
- **THEN** 提示用户"内容过少，无法有效蒸馏"

#### 需求：纯文本蒸馏
系统 SHALL 支持用户直接粘贴文本进行蒸馏。

**场景：文本蒸馏**
- **WHEN** 用户在 Web 端粘贴文本（≥ 100 字）并点击蒸馏
- **THEN** 系统调用 AI 生成结构化总结

#### 需求：抖音视频蒸馏
系统 SHALL 支持用户粘贴抖音视频分享链接，自动下载视频、分离音频、识别文案并蒸馏为知识。

**场景：正常抖音视频蒸馏**
- **WHEN** 用户粘贴抖音分享链接（如 `https://v.douyin.com/xxxxx/`）并点击蒸馏
- **THEN** 系统通过 Python 服务下载视频 → 提取音频 → ASR 识别为文案 → 调用 AI 蒸馏，60 秒内返回结构化结果

**场景：抖音直播切片/长视频**
- **WHEN** 视频时长 > 10 分钟
- **THEN** 系统分块处理音频，合并识别结果后再蒸馏，总耗时 < 3 分钟

**场景：抖音链接无效**
- **WHEN** 链接无法解析或视频已被删除
- **THEN** 提示"无法获取视频，请检查链接或视频是否已被删除"

**场景：抖音视频无语音**
- **WHEN** 视频为纯音乐/无声视频
- **THEN** 提示"该视频无有效语音内容，无法蒸馏"

#### 需求：小红书笔记蒸馏
系统 SHALL 支持用户粘贴小红书笔记链接，自动提取图文内容或视频内容并蒸馏。

**场景：小红书图文笔记蒸馏**
- **WHEN** 用户粘贴小红书图文笔记链接并点击蒸馏
- **THEN** 系统通过 Python 服务提取笔记标题、正文、图片 OCR 文本，合并后调用 AI 蒸馏

**场景：小红书视频笔记蒸馏**
- **WHEN** 用户粘贴小红书视频笔记链接并点击蒸馏
- **THEN** 系统下载视频 → 分离音频 → ASR 识别 → 蒸馏，流程同抖音

**场景：小红书链接无效**
- **WHEN** 链接无法解析或笔记已被删除/设为私密
- **THEN** 提示"无法获取笔记内容，请检查链接或笔记是否已公开"

#### 需求：PDF 文件蒸馏（基于 opendataloader-pdf）
系统 SHALL 使用 opendataloader-pdf 引擎上传 PDF 文件并进行高精度蒸馏，支持复杂表格、扫描件 OCR、数学公式。

**场景：上传标准 PDF**
- **WHEN** 用户上传 PDF 文件（≤ 20MB）
- **THEN** 系统通过 Python 服务调用 opendataloader-pdf 提取 Markdown，分块蒸馏，30 秒内返回结构化结果

**场景：上传扫描件 PDF**
- **WHEN** PDF 为扫描件（无文本层）
- **THEN** 系统启用 opendataloader-pdf Hybrid 模式 + OCR，提取文本后蒸馏

**场景：PDF 过大**
- **WHEN** 用户上传超过 20MB 的 PDF
- **THEN** 提示"文件过大，请上传 20MB 以内的文件"

**场景：PDF 包含复杂表格**
- **WHEN** PDF 包含多级嵌套表格
- **THEN** opendataloader-pdf Hybrid 模式正确解析表格结构，蒸馏结果保留表格数据

### 5.4 知识库管理

#### 需求：知识条目 CRUD
系统 SHALL 提供知识条目的创建、查看、编辑、删除功能。

**场景：查看列表**
- **WHEN** 用户进入知识库页面
- **THEN** 显示所有知识条目，支持卡片/列表视图切换，按创建时间倒序

**场景：查看详情**
- **WHEN** 用户点击某条目
- **THEN** 显示完整详情：标题、摘要、关键点、大纲、标签、原文、关联知识

**场景：编辑条目**
- **WHEN** 用户点击编辑
- **THEN** 可修改标题、标签、补充个人笔记

**场景：删除条目**
- **WHEN** 用户点击删除并确认
- **THEN** 条目软删除，30 天内可恢复

#### 需求：标签管理
系统 SHALL 支持标签的创建、着色、过滤。

**场景：创建标签**
- **WHEN** 用户在蒸馏结果或详情页输入新标签名
- **THEN** 创建标签并关联到当前知识

**场景：按标签过滤**
- **WHEN** 用户在知识库点击某标签
- **THEN** 仅显示带有该标签的知识条目

#### 需求：全文搜索
系统 SHALL 支持对知识库的全文搜索。

**场景：关键词搜索**
- **WHEN** 用户在搜索框输入关键词
- **THEN** 返回标题、摘要、原文中包含关键词的条目，关键词高亮

**场景：组合过滤**
- **WHEN** 用户同时选择标签 + 时间范围 + 来源类型
- **THEN** 返回满足所有条件的条目

### 5.5 知识图谱

#### 需求：图谱自动构建
系统 SHALL 在蒸馏时自动抽取实体并建立知识关联。

**场景：实体抽取**
- **WHEN** 蒸馏完成
- **THEN** 从内容中抽取实体（人名/概念/组织/技术），存入数据库

**场景：自动关联**
- **WHEN** 新知识入库
- **THEN** 与已有知识基于实体重叠 + 标签相似度建立关联，关联强度 0-1

#### 需求：图谱可视化
系统 SHALL 提供交互式的知识图谱可视化。

**场景：查看图谱**
- **WHEN** 用户进入图谱页面
- **THEN** 以力导向布局展示所有知识节点和关联边

**场景：节点交互**
- **WHEN** 用户点击某节点
- **THEN** 跳转到该知识详情页

**场景：图谱过滤**
- **WHEN** 用户选择标签或时间范围
- **THEN** 图谱仅显示符合条件的节点和关联边

## 六、非功能需求

### 6.1 性能
- 单篇网页蒸馏响应时间：< 10 秒
- 50 页 PDF 蒸馏响应时间：< 30 秒（使用 opendataloader-pdf）
- 抖音/小红书短视频（< 5 分钟）蒸馏响应时间：< 60 秒
- 抖音/小红书长视频（> 10 分钟）蒸馏响应时间：< 3 分钟
- 知识库列表加载：< 500ms
- 全文搜索响应：< 200ms
- 图谱渲染（100 节点）：< 1 秒
- Python 媒体处理服务与 Next.js 主服务通过 HTTP 异步通信，不阻塞主线程

### 6.2 可用性
- Web 端响应式设计，支持桌面端（≥ 1024px）和移动端浏览器（≥ 375px）
- 浏览器插件支持 Chrome/Edge（Chromium 内核）
- 关键操作有明确的加载态、空状态、错误态
- 视频/PDF 处理为耗时操作，需提供进度反馈（轮询或 SSE）

### 6.3 安全性
- 用户密码使用 bcrypt 哈希存储
- LLM API Key 使用 AES-256-GCM 加密存储
- 所有 API 接口需认证（除登录注册）
- 用户数据隔离，A 用户无法访问 B 用户数据
- 输入校验防止 SQL 注入、XSS
- Python 服务仅接受内部网络调用，不对外暴露
- 下载的视频/音频文件处理完成后立即删除，不长期存储
- 抖音/小红书链接处理需遵守平台 robots.txt 与用户协议

### 6.4 可靠性
- LLM API 调用失败时自动重试（最多 3 次）
- 蒸馏任务失败时保留原始内容，允许重试
- 数据库定期备份（生产环境）
- Python 媒体处理服务崩溃时主服务可降级（仅支持文本/网页蒸馏）
- 视频下载失败时提供清晰错误提示

## 七、成功指标

### 7.1 MVP 验证指标
- 用户可完成完整流程：注册 → 配置 API → 蒸馏 → 查看知识库 → 查看图谱
- 单次蒸馏成功率 ≥ 95%
- 平均蒸馏耗时 ≤ 10 秒（网页）/ 30 秒（PDF）

### 7.2 用户体验指标
- 首次蒸馏完成率 ≥ 80%
- 7 日留存率 ≥ 40%
- 用户平均蒸馏次数 ≥ 5 次/周

## 八、约束与假设

### 8.1 约束
- 用户需自行注册并配置 LLM API Key（产品不内置）
- 浏览器插件仅支持 Chromium 内核浏览器
- PDF 文件大小限制 20MB
- 单次蒸馏文本长度限制 10 万字
- 抖音/小红书视频时长限制 30 分钟（更长视频需手动裁剪）
- Python 媒体处理服务需独立部署（需 Java 11+ 与 Python 3.10+，opendataloader-pdf 依赖）
- ASR 识别使用云端 API（如通义听悟/讯飞/Whisper API），需用户配置或使用默认供应商
- 视频下载受平台反爬限制，可能需要配置 Cookie

### 8.2 假设
- 用户拥有至少一个 LLM 供应商的 API Key
- 部署环境支持 PostgreSQL 数据库
- 用户主要使用中文内容（AI Prompt 与 ASR 优化为中文场景）
- 部署服务器可安装 Java 11+ 运行时（opendataloader-pdf 依赖）
- 抖音/小红书分享链接为公开可访问内容
- ASR 服务可用（用户配置 LLM 供应商中至少一家支持语音，或单独配置 ASR 服务）

## 九、术语表

| 术语 | 定义 |
|------|------|
| 蒸馏 | 通过 AI 从原始内容中提取核心要点并结构化的过程 |
| 知识条目 | 一次蒸馏产生的完整知识单元（含原文、摘要、大纲等） |
| 实体 | 从内容中抽取的具体对象（人名、概念、组织、技术等） |
| 知识图谱 | 知识条目之间基于关联关系形成的可视化网络 |
| 关联强度 | 两个知识条目之间关联的紧密程度，0-1 之间的数值 |
| ASR | Automatic Speech Recognition，自动语音识别，将音频转为文本 |
| opendataloader-pdf | 开源 PDF 解析引擎，支持复杂表格、OCR、公式提取，Apache 2.0 协议 |
| 媒体处理服务 | Python 编写的独立微服务，负责视频下载、音频分离、ASR、PDF 解析 |
